포인팅 효과 시뮬레이터 구현 내용 정리

1. 구현 목적

이 시뮬레이터는 Zurlo et al. (2020)의 포인팅 효과(Poynting effect) 논문에서 다룬 비압축성 고무 원기둥의 비틀림-신장 거동을 계산하고 시각화하기 위해 구현하였다. 원기둥에 축방향 하중이 작용하고, 동시에 비틀림 또는 토크가 가해질 때 평형 높이 h, 평형 반지름 r, 늘어난 길이 h - H가 어떻게 변하는지를 확인할 수 있도록 만들었다.

구현은 두 가지 형태로 구성하였다.

- 웹 시뮬레이터: poynting-web/index.html, poynting-web/app.js, poynting-web/styles.css
- MATLAB 시뮬레이터: poynting_simulation.m


2. 입력 변수

최종 UI에서는 실제 물리 모델에 필요한 핵심 입력 변수만 남겼다.

- 초기 높이 H
  - 원기둥의 변형 전 높이이다.
  - 단위는 mm로 입력하며, 내부 계산에서는 m로 변환한다.

- 초기 반지름 Re
  - 원기둥의 변형 전 반지름이다.
  - 논문에서는 직경 8 mm 조건을 사용하므로 기본값은 Re = 4 mm이다.

- 전단 탄성률 mu
  - 고무 재료의 전단 탄성률이다.
  - 논문 조건에 맞춰 기본값을 400 psi로 설정하였다.
  - 계산에서는 1 psi = 6894.76 Pa를 사용해 SI 단위로 변환한다.

- 매달린 질량 m
  - 원기둥 아래에 매달린 질량이다.
  - 축방향 하중은 F = mg로 계산한다.
  - 기본값은 m = 0.300 kg이고, g = 9.81 m/s^2를 사용한다.

초기에 존재했던 "최대 비틀림"과 "최대 토크" 입력은 제거하였다. 이 값들은 논문의 물리 모델을 결정하는 독립 입력 변수라기보다 그래프 범위와 애니메이션 범위를 정하는 보조 설정이었기 때문이다. 현재 비틀림 또는 현재 토크는 3D 시각화 아래의 제어 막대로 직접 조절하도록 정리하였다.


3. 계산 모드

계산 모드는 두 가지로 구현하였다.

첫 번째는 비틀림 각도 지정 모드이다.

이 모드에서는 사용자가 현재 비틀림 값을 theta / 2pi, 즉 회전 수(turns)로 조절한다. 내부적으로는

theta = 2 pi (turns)

로 변환한 뒤 논문에서 제시된 하중 + 비틀림 평형 방정식을 풀어 높이를 계산한다.

두 번째는 토크 지정 모드이다.

이 모드에서는 사용자가 현재 토크 M을 조절한다. 논문 Sec. VII의 torque-controlled problem을 반영하여, 주어진 토크에서 평형 높이 h와 그에 대응하는 비틀림 각도 theta를 동시에 계산한다.


4. 무하중 모드 제거

초기 버전에는 "무하중"과 "하중 + 비틀림" 모드가 따로 있었지만, 최종 버전에서는 무하중 모드를 별도 계산 모드로 두지 않았다.

그 이유는 매달린 질량 m을 0으로 두면 F = mg = 0이 되므로, 별도의 무하중 모드 없이도 무하중 조건을 그대로 표현할 수 있기 때문이다. 실제로 코드 검증에서 m = 0일 때 논문의 무하중 식 Eq. 13과 하중 포함 식 Eq. 15의 결과가 일치하는 것을 확인하였다.

따라서 최종 계산 모드는 "비틀림"과 "토크"만 남겼다.


5. 반영한 논문 식

시뮬레이터에는 논문에서 중요한 식을 모두 반영하였다.

5.1 무하중 비틀림 식

무하중 조건에서 비틀림에 의해 원기둥 높이가 변하는 식은 다음과 같다.

h = H (1 + Re^2 theta^2 / 4H^2)^(1/3)

이 식은 논문의 Eq. 13에 해당한다. 현재 UI에는 별도 무하중 모드를 두지 않았지만, 내부 계산에서는 비교 및 검증용으로 유지하였다.

5.2 가늘고 긴 원기둥 근사식

가늘고 긴 원기둥 조건에서의 근사식도 계산하였다.

h - H ≈ Re^2 theta^2 / 12H

이 식은 논문의 Eq. 14에 해당한다. 최종 화면 그래프에서는 혼란을 줄이기 위해 표시하지 않지만, 코드 내부에서는 근사값 비교가 가능하도록 계산한다.

5.3 하중과 비틀림이 동시에 있는 경우

매달린 질량에 의한 하중 F = mg와 비틀림 theta가 동시에 작용할 때는 다음의 3차 방정식을 풀도록 구현하였다.

(h/H)^3 - [F / (mu pi Re^2)] (h/H)^2 - [1 + Re^2 theta^2 / 4H^2] = 0

이는 논문의 Eq. 15에 해당한다. 웹과 MATLAB 모두 같은 식을 사용한다.

5.4 토크 지정 문제

논문 Sec. VII에서 다루는 토크 지정 문제도 구현하였다. 토크 M이 주어졌을 때의 평형 방정식은 다음과 같다.

(h/H)^3 - [F/(mu pi Re^2) + M^2/(mu^2 pi^2 Re^6)](h/H)^2 - 1 = 0

그리고 계산된 높이 h와 토크 M으로부터 비틀림 각도는 다음 식으로 계산한다.

theta = 2hM / (mu pi Re^4)

이 기능을 통해 사용자가 토크를 직접 조절했을 때 원기둥이 얼마나 비틀리고 늘어나는지 확인할 수 있다.


6. 수치 계산 방식

높이 h는 직접 근사식만 사용하는 것이 아니라, 논문에서 제시한 평형 3차 방정식을 수치적으로 풀어 계산한다.

코드에서는 h/H = lambda로 두고 양의 실근을 찾아 평형 높이를 계산한다. 이후 비압축성 조건에 따라 부피가 보존되므로 평형 반지름 r은 다음 관계로 계산한다.

pi Re^2 H = pi r^2 h

따라서

r = Re sqrt(H / h)

로 계산한다.

이 때문에 비틀림 또는 하중으로 높이가 증가하면 반지름은 약간 감소한다.


7. 출력값

시뮬레이터는 다음 값을 출력하도록 구현하였다.

- 평형 높이 h
- 평형 반지름 r
- 전체 신장 h - H
- 비틀림 추가 신장 h(theta) - h(0)
- 상대 높이 h / H
- 초기 부피
- 현재 부피
- 부피비
- 하중 F = mg
- 토크 M

물리적 표현 오류를 줄이기 위해 "질량"과 "하중"을 구분하였다. 사용자가 입력하는 값은 질량 m이고, 실제 축방향 힘은 F = mg로 계산하여 "하중 F = mg"로 표시한다.

특히 신장량은 두 기준을 구분하도록 수정하였다. h - H는 변형 전 stress-free 길이 H를 기준으로 한 전체 신장량이다. 반면 h(theta) - h(0)는 같은 하중을 먼저 가한 뒤, 비틀림 때문에 추가로 늘어난 길이만 나타낸다. 실험에서 무게를 단 뒤의 위치를 기준으로 비틀림에 의한 추가 이동량을 측정했다면 h(theta) - h(0)를 비교해야 한다.


8. 3D 시각화 구현

3D 시각화에서는 원기둥이 비틀리면서 높이가 증가하고 반지름이 감소하는 모습을 보여준다. 웹 버전에서는 Canvas 기반으로 원기둥 표면, 나선 기준점, 비틀림 방향, 높이 눈금을 직접 그렸다.

초기에는 반지름이 커질 때 화면 카메라 범위도 같이 커져서, 실제로는 반지름이 증가해도 화면에서는 오히려 작아 보이는 문제가 있었다. 이를 수정하기 위해 시각화용 x-y 기준 프레임을 고정하고, 입력 반지름 Re가 커지면 화면에서도 원기둥이 명확히 두꺼워지도록 변경하였다.

또한 "반지름 몇 배 표시" 같은 배율 문구는 사용자에게 혼란을 줄 수 있어 제거하고, 현재 초기 반지름 Re 값을 mm 단위로 표시하도록 정리하였다.


9. 그래프 구현

초기 그래프는 늘어난 길이, 상대 높이, 반지름을 모두 표시하고 Eq. 13, Eq. 14, Eq. 15를 함께 비교했기 때문에 의미가 복잡하게 보였다.

최종 버전에서는 그래프를 두 기준으로 분리하여 표시한다.

- 첫 번째 그래프: 전체 신장 h - H
- 두 번째 그래프: 비틀림 추가 신장 h(theta) - h(0)

질량이 0이 아니면 h - H 그래프는 0 turns에서도 이미 양수로 시작한다. 이는 비틀림 때문이 아니라 매달린 질량에 의한 하중 신장이다. 포인팅 효과 자체, 즉 비틀림으로 인한 추가 신장을 보려면 두 번째 그래프 h(theta) - h(0)를 확인해야 한다. 노란 점은 현재 제어 막대가 가리키는 값을 나타낸다.


10. 애니메이션 구현

애니메이션은 별도의 물리 입력값을 새로 받지 않고, 현재 선택된 계산 모드의 제어 막대를 0에서 기준 끝값까지 자동으로 움직이는 방식으로 구현하였다.

- 비틀림 모드에서는 0 turns에서 6 turns까지 움직인다.
- 토크 모드에서는 0 N m에서 0.04143536 N m까지 움직인다.

토크 모드의 0.04143536 N m는 논문 기본 조건에서 6 turns에 대응하는 토크값이다. 따라서 비틀림 지정 모드와 토크 지정 모드가 같은 최종 상태를 재현하는지 확인할 수 있다.


11. 논문 실험값과의 비교 검증

논문 조건에 해당하는 기본값은 다음과 같이 설정하였다.

- H = 1.0 m
- Re = 4.0 mm
- mu = 400 psi
- m = 0.300 kg
- theta / 2pi = 6 turns

이 조건에서 계산된 값은 다음과 같다.

- 하중만 있을 때 늘어난 길이: 7.1268 mm
- 6 turns 비틀림까지 포함했을 때 총 늘어난 길이: 9.0181 mm
- 비틀림에 의한 추가 신장량: 1.8913 mm
- 6 turns에 대응하는 토크: 0.04143536 N m

논문에서 보고된 값은 약 9 mm이므로, 시뮬레이션 결과 9.0181 mm와 매우 잘 일치한다. 보고값 9 mm를 기준으로 하면 차이는 약 0.0181 mm이고, 상대 오차는 약 0.20%이다.

보고서에서는 비교 기준을 명확히 구분해야 한다. 논문 Eq. 15의 전체 높이 예측과 비교할 때는 h - H = 9.0181 mm를 사용한다. 그러나 실험에서 무게를 단 뒤의 초기 위치를 기준으로 비틀어서 추가로 내려간 거리를 비교한다면 h(6) - h(0) = 1.8913 mm를 사용해야 한다.

또한 토크 지정 모드에서 M = 0.04143536 N m를 입력하면 6 turns와 같은 최종 높이를 얻는 것을 확인하였다. 따라서 비틀림 지정 방식과 토크 지정 방식이 서로 일관되게 구현되었음을 확인하였다.


12. 구현에서 제외하거나 제거한 요소

최종 UI에서는 다음 요소를 제거하였다.

- 별도의 무하중 계산 모드
  - m = 0으로 두면 무하중 조건이 되므로 별도 모드가 필요하지 않다.

- 입력 변수 영역의 최대 비틀림
  - 실제 물리 변수라기보다 제어 막대 범위였기 때문에 제거하였다.

- 입력 변수 영역의 최대 토크
  - 실제 물리 변수라기보다 토크 제어 막대 범위였기 때문에 제거하였다.

- 변형 전 기준 원기둥 표시 옵션
  - 필수 계산 기능이 아니고 UI를 복잡하게 만들어 제거하였다.

- 표시 옵션 변경 상태 표시
  - 계산 결과와 직접 관련이 없어 제거하였다.

- 복잡한 3개 그래프 표시
  - 레포트와 사용자 이해를 위해 h - H 그래프 하나로 단순화하였다.

- CSV 저장 및 PNG 그림 저장
  - 보고서 작성에 직접 필요하지 않고 UI를 복잡하게 만들어 제거하였다.


13. 최종 구현 요약

최종적으로 구현된 시뮬레이터는 논문에서 중요한 변수 H, Re, mu, m을 입력받고, 하중 F = mg와 비틀림 또는 토크에 따른 원기둥의 평형 상태를 계산한다. 계산에는 논문의 Eq. 13, Eq. 14, Eq. 15와 토크 지정 방정식을 반영하였다. 결과는 3D 시각화, 전체 신장 h - H 그래프, 비틀림 추가 신장 h(theta) - h(0) 그래프, 수치 출력으로 확인할 수 있다.

검증 결과, 논문 기본 조건에서 계산된 최종 신장량은 9.0181 mm로 논문의 약 9 mm 결과와 잘 일치하였다. 따라서 구현된 시뮬레이션은 논문의 해석 결과와 실험적 관찰을 정량적으로 재현한다고 볼 수 있다.
